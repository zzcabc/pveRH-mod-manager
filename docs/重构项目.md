# 重构mod管理器

## 基础配置

1. 在程序执行目录下生成 `config.json`内容如下

```json
{
  "game_path": [
    {
      "path": "E:\\Game\\植物大战僵尸融合版3.7Mod",
      "version": "3.7"
    },
    {
      "path": "E:\\Game\\植物大战僵尸融合版3.6.1Mod",
      "version": "3.6.1"
    }
  ],
  "mod_path": [
    "E:\\Game\\MOD"
  ],
  "download_path": ""
}
```

2. 可以选择多个游戏路径，版本非必填，在前端根据mod文件夹查询版本有哪些
3. `download_path` 由 `admin.html`中选择



4. 以下为`index.html`大概内容

|            |                                       |
|------------|---------------------------------------|
| 游戏路径       | 游戏版本                                  |
| MOD路径      | MOD版本                                 |
| 检测BepInEx  | (如果已安装显示`卸载BepInEx`,未安装显示`安装BepInEx`) |
| `安装高数修改器`  | `安装梧萱修改器`                             |
| `启动游戏`     ||
| mod名称模糊查询框 |                                       |
| 植物MOD(未安装) | 植物MOD(已安装) [移除已安装mod]                 |
|            |                                       |
| 僵尸MOD(未安装) | 僵尸MOD(已安装) [移除已安装mod]                 |
|            |                                       |
| 皮肤(未安装)    | 皮肤(已安装)                               |
| 在线MOD管理    |                                       |
|            | 植物MOD                                 |
|            | 僵尸MOD                                 |
|            | 皮肤                                    |
|            | 其他                                    |

## 添加检测`BepInEx`

1. 如果已安装(查看第二点)，显示状态
2. 读取项目内的`gamefile.json`中`BepInEx`内容 , 查询游戏目录下有没有这些内容，如果存在则为已安装，否则为未安装
3. 点击`安装BepInEx`查询`BepInEx.zip` 解压到游戏目录下
4. 点击`移除BepInEx`保留`gamefile.json` 中`GameFile`内容，移除其他文件或文件夹

## 添加安装修改器 

点击安装`安装高数修改器`, 检查mod文件夹有没有 类似`PvZRHModfiedFor(版本)[].zip`格式的压缩包，记得匹配版本，如果有解压到游戏目录下，如果没有查询网站这个文件(查询功能暂时不做，等待服务器程序)

## 启动游戏

游戏目录下启动`PlantsVsZombiesRH.exe`

## MOD管理
1. 未安装mod为mod文件夹内的mod，`mod.dll`查询`mod.json`的 `modName` 返回 `chineseName`
2. 已安装mod为游戏目录下`BepInEx/plugins`文件夹内的mod，`mod.dll`查询`mod.json`的 `modName` 返回 `chineseName`
3. 点击`移除已安装mod`，移除游戏目录下`BepInEx/plugins`文件夹内的mod
4. 僵尸同上
