# 重构规划

## 一、配置结构重构

### 1.1 目标配置格式

```json
{
  "game_path": [
    {
      "path": "E:\\Game\\植物大战僵尸融合版3.7Mod",
      "version": "3.7"
    }
  ],
  "mod_path": [
    "E:\\Game\\MOD"
  ],
  "download_path": ""
}
```

### 1.2 实现步骤

| 步骤 | 内容 | 验证方式 |
|------|------|----------|
| 1 | 定义新的 `GamePath` 结构体（含 `path` 和 `version` 字段） | 编译通过 |
| 2 | 修改 `Config` 结构体：`GamePaths` → `GamePath []GamePath`，`ModLibPaths` → `ModPath []string` | 编译通过 |
| 3 | 实现版本自动检测：扫描 MOD 目录下的子文件夹，提取版本号（如 `3.7`、`3.6.1`） | 单元测试 |
| 4 | 修改 `loadConfig()` 和 `saveConfig()` 适配新结构 | 启动后 config.json 格式正确 |
| 5 | 添加兼容逻辑：读取旧格式配置时自动转换为新格式 | 旧配置文件可正常加载 |

### 1.3 版本检测逻辑

```
MOD目录结构示例：
E:\Game\MOD\
├── 慕容孤晴-3.7\
├── 慕容孤晴-3.6.1\
├── 高数羽衫-3.7\
└── BepInEx.zip

检测方式：
1. 遍历 MOD 目录下的子文件夹
2. 使用正则匹配文件夹名中的版本号：\d+\.\d+(\.\d+)?
3. 去重后返回所有可用版本
```

## 二、mod.json 集成

### 2.1 数据结构

项目内的 `mod.json` 格式：
```json
[
  {
    "author": "作者名",
    "version": "3.7",
    "planList": [{"chineseName": "中文名", "modName": "xxx.dll"}],
    "zombieList": [...],
    "skinList": [...],
    "pluginList": null
  }
]
```

### 2.2 实现步骤

| 步骤 | 内容 | 验证方式 |

## 三、MOD 分类管理

### 3.1 分类依据

根据 mod.json 中的字段分类：
- `planList` → 植物 MOD
- `zombieList` → 僵尸 MOD
- `skinList` → 皮肤（暂不实现安装/卸载）
- `pluginList` → 插件（暂无数据）

### 3.2 实现步骤

| 步骤 | 内容 | 验证方式 |

## 九、风险和注意事项

1. **向后兼容**：旧配置文件需要能自动转换
2. **版本匹配**：MOD 安装时需要匹配当前选择的版本
3. **文件锁定**：Windows 下文件被占用时的处理
4. **错误处理**：所有操作需要有明确的错误提示
5. **日志记录**：关键操作需要记录日志便于排查问题
