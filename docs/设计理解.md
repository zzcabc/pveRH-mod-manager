## 1. 项目定位

**Windows 桌面 GUI 程序**，用于管理《植物大战僵尸融合版》的 BepInEx 框架及各种 MOD。

- 语言：Go 1.25
- 目标平台：Windows
- 游戏目录：`D:\Game\植物大战僵尸融合版3.7Mod`
- MOD 目录：`D:\Game\PVERH-MOD`

## 3. 数据结构

### 3.1 config.json

```json
{
  "game_path": [
    { "path": "D:\\Game\\植物大战僵尸融合版3.7Mod", "version": "3.7" },
    { "path": "D:\\Game\\植物大战僵尸融合版3.6.1Mod", "version": "3.6.1" }
  ],
  "mod_path": [
    "D:\\Game\\PVERH-MOD"
  ],
  "download_path": ""
}
```

- `game_path`：数组，每项绑定 `path` 和 `version`
- `mod_path`：数组，UI 下拉框切换
- `download_path`：下载目录（暂未关联 UI）

### 3.2 gamefile.json

```json
{
  "GameFile": ["PlantsVsZombiesRH_Data/", "baselib.dll", "..."],
  "BepInEx": ["BepInEx/", "dotnet/", ".doorstop_version", "..."]
}
```

- `GameFile`：游戏本体文件列表，卸载 BepInEx 时的「白名单」
- `BepInEx`：BepInEx 框架文件列表，用于检测安装状态 + 卸载时的删除清单

### 3.3 Go 结构体设计

```go

    Version string `json:"version"`
```

## 4. 功能模块拆解

### 4.1 配置管理 (`config.go`)

- 启动时读取/生成 `config.json`
- 首次运行无 `config.json` 时创建默认空模板
- `game_path.version` 通过扫描 MOD 目录中的版本号确定
- `mod_path` 支持多目录，UI 下拉切换

### 4.2 BepInEx 模块 (`bepinex.go`)

**检测**：
- 读取 `gamefile.json` 的 `BepInEx` 列表
- 逐条检查游戏目录下是否存在
- 全部存在 → `已安装`，否则 → `未安装`

**安装**：
- 查找 `BepInEx.zip`（优先 MOD 目录，其次程序目录）
- 解压到当前选中的游戏目录
- 覆盖已有文件

**卸载**：
- 读取 `gamefile.json` 的 `GameFile`（白名单）和 `BepInEx`（删除清单）
- 删除 `BepInEx` 中的所有文件/目录
- 保留 `GameFile` 中的文件/目录
- **注意**：目录类型的条目需递归删除（如 `BepInEx/`、`dotnet/`）

### 4.3 修改器安装 (`modifier.go`)

- 扫描 MOD 目录中匹配 `PvZRHModfiedFor{版本}[].zip` 的压缩包
  - `{版本}` 来自 `config.json` 当前选中游戏的 `version`
  - `[]` 为可选内容（如日期标注）
- 正则：`PvZRHModfiedFor(\d+\.\d+(?:\.\d+)?).*\.zip`
- 匹配到 → 解压到游戏目录
- 未匹配到 → 提示未找到（网络查询功能后续接入）

### 4.4 MOD 管理 (`mod.go`)

**MOD 类型**：植物MOD、僵尸MOD、皮肤 — 逻辑相同，按目录区分。

**未安装列表**：
- 扫描 MOD 目录下对应子目录
- 每个子目录内找 `mod.dll`
- 显示名取目录名（`mod.json` 已废弃）
- 点击 → 复制到 `BepInEx/plugins/` → 变为已安装

**已安装列表**：
- 扫描 `BepInEx/plugins/` 目录
- 显示名取目录名
- 提供两个操作：
  - **单个移除**：删除该 mod 的 dll 及相关文件
  - **全部移除**：清空 `BepInEx/plugins/` 下所有内容

### 4.5 UI 层 (`ui.go` 或 GUI 框架对应代码)

按页面配置 9 行布局：

```
[游戏目录: 下拉框 ▼] [版本: 下拉框 ▼]
[MOD目录: 下拉框 ▼]
[BepInEx: 已安装/未安装] [安装BepInEx / 卸载BepInEx]
[安装高数修改器] [安装梧萱修改器]
[打开游戏目录] [打开MOD目录] [打开已安装目录]
[植物MOD(未安装)] [植物MOD(已安装)]
[僵尸MOD(未安装)] [僵尸MOD(已安装)]
[皮肤(未安装)]   [皮肤(已安装)]
[在线MOD]
```

- 状态按钮文本随状态切换（已安装→卸载 / 未安装→安装）
- 版本选择框随当前游戏目录联动

## 5. 关键业务规则

| 规则 | 说明 |
|---|---|
| **版本判断** | 从 MOD 目录中存在的修改器压缩包文件名提取版本号 |
| **白名单卸载** | 卸载 BepInEx 时，`GameFile` 中的条目必须保留 |
| **目录匹配** | `gamefile.json` 中路径末尾 `带/` = 目录，需递归操作 |
| **MOD 识别** | MOD 通过 `mod.dll` 存在性判断，显示名取所在目录名 |
| **修改器版本匹配** | 必须匹配当前游戏版本，不跨版本安装 |
| **多 MOD 目录** | 支持，UI 下拉切换，所有扫描操作限定在当前选中的目录 |
